UPDATE / INSTALL STEPS
1. BACK UP your current working website files and Apps Script code.
2. Open the Google Apps Script project connected to the Kawaii Aqua Pets Google Sheet.
3. Replace the existing Code.gs with the Code.gs in this folder.
4. Save.
5. Run setupStore() ONCE.
   - It adds missing columns if needed.
   - It creates/normalizes the four demo categories.
   - It adds missing sample products.
   - It adds missing DEMO REVIEW rows.
   - It does not intentionally delete your existing Orders, real Products, real Reviews, or Settings.
6. Approve Google permissions if prompted.
7. Deploy > Manage deployments > Edit > New version > Deploy.
8. Keep Execute as: Me and the public access setting required by your current store.
9. Copy the latest Apps Script Web App URL.
10. Open config.js and paste your CURRENT WORKING Web App URL into window.STORE_API_URL.
11. Replace/upload these GitHub website files:
    - index.html
    - admin.html
    - app.js
    - admin.js
    - app.css
    - config.js
12. Keep/upload the assets folder.
13. Hard refresh the customer and admin websites after GitHub Pages updates.

LOCAL DEMO PREVIEW
If config.js still contains:
PASTE_YOUR_GOOGLE_APPS_SCRIPT_WEB_APP_URL_HERE

the customer page automatically opens in DEMO PREVIEW mode and displays:
- 20 sample products
- 4 categories
- 18 DEMO REVIEW cards

Add to Cart and Buy Now can be previewed, but live order/review submission is blocked until Apps Script is connected.

LIVE GOOGLE SHEET SEED DATA
Run setupStore() once after replacing Code.gs.
The seeded products and demo reviews then come from the Google Sheet and can be viewed/managed in Admin.

REMOVE DEMO PRODUCTS AND REVIEWS LATER
In Apps Script, manually run:
removeDemoStoreData()

This removes rows whose IDs start with:
- DEMO_PRD_
- REV_DEMO_

It does not delete the Betta, Guppies, Snails, or Shrimps categories.

ADMIN ORDER FLOW
Pending > Paid > Preparing > Ready > Shipped > Completed

- Paid: deducts stock and sends the payment receipt email.
- Before Shipped: open Shipping Details > Customer Order Tracking.
  - LBC: enter the LBC tracking number. Tracking page URL is optional.
  - Lalamove: paste the Lalamove tracking/share link.
- Shipped: sends the customer a shipping update email.
- Completed: marks the order completed.
- Cancelled: stops the order and restores stock when stock had already been deducted.

CUSTOMER TRACKING
The customer opens Track Order and enters:
- Order ID, for example ORD_...
- Same checkout email OR same mobile number

ADMIN AUTO REFRESH
- Checks for new Orders and Reviews every 15 seconds while the Admin page is visible.
- No browser reload is needed.
- Manual Refresh Data button remains available.
- New order/review notifications use a toast and red count badge.
- Returning to the Admin browser tab triggers an immediate fresh-data check.

SECURITY NOTE
The public tracking action does not return payment proof images or the full customer address.
It only returns allowed order status, item, total, payment/delivery summary, timeline, and saved tracking information after the Order ID + matching email/mobile check.
